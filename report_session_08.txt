## 2. 쿼리 생성 모델 선정 — Qwen2.5-VL-7B-Instruct
================================================================================

  ### 2-1. 모델 탐색 경과

  초기 구현(PaliGemma 2 28B)이 쿼리 생성 태스크에 부적합함을 확인.
  멀티모달 LLM 전환을 결정하고 아래 순서로 평가 진행.

  [평가 모델]
    PaliGemma 2 28B (4-bit)
      → DSV 포맷 불안정, 한국어 생성 품질 저하. 탈락.

    Qwen2.5-VL-7B-Instruct (bfloat16)
      → 건당 ~1.2초, DSV 유효율 90~100%, 한국어 품질 양호. 채택.

    Qwen2.5-VL-32B-Instruct (4-bit)
      → 건당 ~8.7초, DSV 포맷 붕괴(줄바꿈·마크다운 혼입), 유효율 0%.
      → 전체 213K건 기준 ~515시간(21일) 소요 예상. 탈락.

    Qwen2.5-VL-72B-Instruct (4-bit)
      → A100 40GB 단일 GPU OOM. 다운로드 도중 실패. 탈락.

  [선정 이유 — 7B]
    - 포맷 안정성: 10건 테스트 기준 90~100% 유효 DSV
    - 속도: 전체 213K건 기준 약 89시간(3.7일) 내 완료 가능
    - GPU: A100 40GB에서 bfloat16 전체 로드 (~14GB), 안정적


  ### 2-2. 페르소나 구성 결정

  초기 4개 페르소나(Poet / Trendy / Space / Philosopher) 중
  Trendy 품질 문제로 인해 3개로 축소.

  [Trendy 실패 원인]
    - 어휘 시딩(갓생·핫플·취향저격 등 MZ 신조어 목록) 추가해도
      7B 모델이 특정 단어(힙한·힙스터)에 고착되는 현상 반복
    - 근본 원인: 7B 규모에서 한국어 MZ 문화 이해 부족

  [최종 채택 페르소나 — 3종]
    Poet       : 은유·서정적 표현  (예: '푸른 새벽의 파편')
    Space      : 날씨·장소·감각적 질감  (예: '비 오는 날 LP바')
    Philosopher: 인간 본질·심연 테마  (예: '존재의 끝에서')


  ### 2-3. generate_queries.py 리팩토링

  [주요 변경]
    - 모델 백엔드 추상화: --model-type qwen | paligemma 선택 가능
    - Qwen2.5-VL 로더 추가 (bfloat16 / 4-bit 자동 선택)
    - 프롬프트 개선: 3-shot 예시, 한국어 전용 출력 강제, 줄바꿈 금지
    - validate_dsv 강화: 멀티라인·마크다운 혼입 파트 자동 거부
    - repetition_penalty=1.3, no_repeat_ngram_size=4 적용

  [생성 파라미터]
    max_new_tokens=80, do_sample=False
    repetition_penalty=1.3, no_repeat_ngram_size=4
