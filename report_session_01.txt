## 1. 프로젝트 개요 파악

영화·음악·도서 3개 도메인을 통합한 멀티모달 콘텐츠 기반 추천 시스템.
각 콘텐츠의 이미지 + 텍스트 + 사용자 쿼리를 인코딩하여 768차원 벡터로 합산한 뒤
ANN(Approximate Nearest Neighbor) 검색으로 유사 콘텐츠를 추천하는 구조.

모델: DualEncoderModel (TextBlock / ImageBlock / QueryBlock / ContentBlock)
학습: InfoNCE Loss (Stage 1) → KL Divergence (Stage 2) 2단계

## 3. MovieGenre.csv — imdbId 제로패딩

[문제]
  imdbId 및 Imdb Link 컬럼의 IMDB ID가 구형 포맷(최소 1자리~최대 6자리).
  현행 IMDB 표준은 7자리(tt0000001 형식).

[자릿수 분포]
  1자리:  3개   예) tt8        → Edison Kinetoscopic Record of a Sneeze (1894)
  2자리:  2개   예) tt12       → The Arrival of a Train (1896)
  3자리: 12개   예) tt417      → A Trip to the Moon (1902)
  4자리: 124개  예) tt9018     → A Dog's Life (1918)
  5자리: 12,958개  예) tt75314  → Taxi Driver (1976)
  6자리: 14,562개  예) tt114709 → Toy Story (1995)
  7자리: 12,447개  (이미 정상)

[처리]
  zfill(7) 으로 모든 ID를 7자리로 제로패딩.
  imdbId 컬럼과 Imdb Link 컬럼 양쪽 모두 수정.

[결과]
  수정된 행: 27,661개
  최종 저장: data/MovieGenre.csv (40,108개, 인코딩: latin-1)


## 4. Books.csv — 이미지 URL 유효성 검증

[배경]
  멀티모달 모델이 이미지를 직접 사용하므로, 실제 이미지가 없는 행은 학습에 부적합.
  Books.csv의 Image-URL-M 컬럼은 Amazon CDN URL.

[문제]
  Amazon CDN은 존재하지 않는 이미지에도 HTTP 200을 반환하며 플레이스홀더를 전송.
  - 43B GIF  (1×1 px)    : 완전 무효 플레이스홀더 (MD5: ad4b0f606e0f8465bc4c4c170b37e1a3)
  - 2254B JPEG (160×107) : "이미지 없음" 플레이스홀더 (MD5: 52429d0be691b5c0823f37eb5cfeaae5)
  User-Agent 없이는 모든 요청이 403 반환 → User-Agent 헤더 필수.

[검증 방법]
  GET 스트림으로 최대 3,000B 읽기.
  - 3,000B 이상 수신 → 유효
  - 3,000B 미만 수신 → MD5로 알려진 플레이스홀더 여부 확인
  ThreadPoolExecutor (200 workers), 체크포인트 2,000건마다 저장.
  스크립트: scripts/validate_image_urls.py

[결과]
  전체:           271,360개
  유효 (실제 이미지): 149,737개 (55.2%)
  무효 (플레이스홀더): 121,623개 (44.8%)
  소요 시간: 약 15분

  무효 행 인덱스: data/books_invalid_indices.json
  필터링된 파일: data/Books_filtered.csv (149,737개)
