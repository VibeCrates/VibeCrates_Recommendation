## 프로젝트 개요

영화, 음악, 도서 도메인을 대상으로 한 멀티모달 추천 시스템.
콘텐츠 이미지, 콘텐츠 텍스트 설명, 사용자 검색 쿼리를 함께 사용하여 콘텐츠 벡터를
생성하고, ANN(Approximate Nearest Neighbor) 검색으로 추천을 수행하는 구조.

< 모델 아키텍처 (DualEncoderModel) >

  DualEncoderModel
  ├── TextBlock
  │     SBERT (paraphrase-multilingual-mpnet-base-v2) + LoRA (r=16) + MLP → 768dim
  ├── ImageBlock
  │     CLIP Vision Encoder (Bingsu/clip-vit-large-patch14-ko) + MLP → 768dim
  ├── QueryBlock
  │     CLIP Text Encoder (동일 모델) + MLP → 768dim
  └── ContentBlock
        concat(z_image, z_text) [1536dim]
        → Linear → LayerNorm → GELU → Dropout (×3)
        → L2 normalize → 768dim

  - 사전학습 모델 파라미터는 전부 Freeze, LoRA 어댑터와 MLP만 학습
  - fit()/predict()/evaluate() 는 NotImplementedError 로 Trainer에 위임


--------------------------------------------------------------------------------
[src/training/] — 학습 파이프라인
--------------------------------------------------------------------------------

config.py   [완성]
  - TrainingConfig dataclass
  - Stage1: 10 에폭, lr=1e-4, temperature=0.07
  - Stage2: 15 에폭, lr=5e-5
  - LoRA: r=16, alpha=32, dropout=0.05
  - batch_size=32, early_stopping_patience=3

losses.py   [완성]
  - InfoNCELoss : 대칭적 양방향 Cross-Entropy (온도 파라미터 적용)
  - KLDivergenceLoss : 지식 증류용 KL Divergence (log_target=True)

trainer.py  [완성]
  - TwoStageTrainer — 2단계 학습 orchestration
  - Stage 1: LoRA 어댑터 + MLP만 활성화
              텍스트-쿼리, 텍스트-이미지, 이미지-쿼리 3쌍 InfoNCE Loss 합산
  - Stage 2: ContentBlock만 활성화
              z_query(teacher, no_grad) vs z_content(student) KL Divergence 최소화

history.py  [완성]
  - TrainingHistory — 배치/에폭 단위 메트릭 기록
  - Stage1: loss_tq, loss_ti, loss_iq, total_loss, val_loss 추적
  - Stage2: kl_loss, val_loss 추적
  - JSON 저장/로드 지원
